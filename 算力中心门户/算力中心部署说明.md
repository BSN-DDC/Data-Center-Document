使其内核参数即刻生效

```bash
sysctl -p
```

关闭透明巨页内存配置

临时生效

```bash
sudo echo never > /sys/kernel/mm/transparent_hugepage/enabled
```

永久生效，加入开机自启动

```bash
sudo echo "echo never > /sys/kernel/mm/transparent_hugepage/enabled" >> /etc/rc.local
```

赋予执行权限

```bash
sudo chmod +x /etc/rc.local
```

### Nginx 配置文件部分配置说明

conf/nginx.conf 配置文件请保持默认即可

请修改 stream 目录下的 operation.conf、portal.conf、openapi.conf、opbgateway.conf 配置文件

如配置证书启动 <strong>80 端口重定向 </strong>功能，否则通过请通过 <strong>域名 + 端口号 </strong>的方式访问对应网页

Nginx 配置文件列表说明

✅  文件需要使用

❌  文件不需要使用

 }

### nginx 端口说明

<strong>80  443</strong> 端口由于部分运营商限制需要进行备案后方可访问，如未备案请修改为其他端口

Nginx 配置文件同样需要进行修改

| 端口号 | 说明                                          |
| ------ | --------------------------------------------- |
| 80     | 配置证书后默认跳转到对应服务的 443 端口       |
| 443    | 配置证书后访问使用端口 https://service.domain |
| 1337   | 代理 kong 节点网关管页面                      |
| 8440   | 运营页面                                      |
| 8441   | 门户页面                                      |
| 8442   | 代理 openapi 服务端口                         |
| 18601  | 代理 kong 节点网关 websocket 端口             |
| 18602  | 代理 kong 节点网关 websockets 端口            |
| 18603  | 代理 kong 节点网关 grpcs 端口                 |
| 18605  | 代理 kong 节点网关 grpc 端口                  |
